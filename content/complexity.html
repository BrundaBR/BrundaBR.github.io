What is Complexity analysis?

Complexity Analysis is measure of performance of an algorithm, it helps to understand the time and Space algorithm would take in order to perform in worst case,we are mainly focused on worst case as it gives us bigger picture of performance of algorithm when huge data is is given has an input,this is reason we use Big O (denoted as O(n) or O(1) so on) as this is the only analysis which gives the worst case factor for us to measure.

An algorithm requires two factors which formulates its efficency which are

  1. Time Complexity
  2. Space Complexity

Time complexity is the amount of computer time taken by an algorithm,
if a function runs with time T1 s is it the time complexity?

NO!
Computer science has different way of describing the time complexity using Big O which is approximate value of complexity where are all constants and calculations are ignored as we really care about is degree of expression if degree is high we say it as worst time complexity and good if it has one as degree and best if it takes constant time which is O(1).

Logarithmic time An algorithm is said to take logarithmic time when T(n) = O(log n). Since loga n and logb n are related by a constant multiplier, and such a multiplier is irrelevant to big-O classification, the standard usage for logarithmic-time algorithms is O(log n) regardless of the base of the logarithm appearing in the expression of T.
