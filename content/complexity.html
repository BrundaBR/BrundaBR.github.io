What is Algorithmic complexity?

Complexity Analysis is measure to analyze performance of an algorithm, it helps to understand the time and Space algorithm would consume for it to run,and basically it!,it is a measure to calcuate efficency of algorithm. We mainly consider Big O (Big O notation is a mathematical notation that describes the limiting behavior of a function when the argument tends towards a particular value or infinity) to know the efficiency as it gives us the worst case measure. Worst case gives us bigger picture regarding performance of algorithm when huge data is is given has an input, this factor or big O notation helps us to figure out efficient way to deal with huge data. This being reason we use Big O notation (denoted as O(n) or O(1) so on) as this is the only analysis which helps program developers to give programmers some basis for computing and measuring the efficiency of a specific algorithm.

An algorithm requires two factors which is considered in order to calcualte its efficency which are as follows

  1. Time Complexity
  2. Space Complexity

Time complexity is the amount of computer time taken by an algorithm,

Computer scientists have different way of describing the efficiency for that matter time complexity using Big O notation which is approximate value of complexity where are all constants and calculations are ignored and degree of an expression is considered to represent efficiency and also solely dependent on input and the way we process input in order to store it. O(1) is considered as Best case.

Constant time does not depend on input, and is a ideal case O(1) in computation which basically pronounced as big O of 1. If T(n) is O(any constant value), this is equivalent to and stated in standard notation as T(n) being O(1).
